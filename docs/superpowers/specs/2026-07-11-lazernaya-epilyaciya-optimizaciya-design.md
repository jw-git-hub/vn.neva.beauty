# Оптимизация страницы «Лазерная эпиляция» — дизайн

Дата: 2026-07-11
Проект: vn.neva.beauty (standalone статика, генератор Python + Jinja2)
Страница: `/lazernaya-epilyaciya/`

## Цель

Применить к странице услуги «Лазерная эпиляция» оба промта оптимизации
(`promt-1-seo.md` + `promt-2-ai-geo.md`) — следующая строка `[ ]` в
`TODO-optimization.md`. Это **первая услуга** после трёх готовых категорий,
поэтому работа делится на структурный слой (общий шаблон услуг → польза всем
10 услугам) и контентный слой (только эта страница).

## Жёсткие ограничения (из промтов)

- **Не менять цены** — ни в текстах, ни в разметке. В schema диапазон выводится
  из реального прайса, не вручную.
- **Не выдумывать факты.** У заказчика **нет** данных по эпиляции (тип лазера,
  число сеансов в курсе, длительность, противопоказания) — подтверждено в этой
  сессии. FAQ и intro остаются **качественными**, без чисел. Только реальные
  факты со страницы (цены, зоны, бренд, город).
- **Не менять URL** страницы.
- Весь пользовательский контент — на русском; технические ключи schema/HTML — по
  стандарту (английский).

## Решения (согласованы с пользователем)

- **Schema цены — вариант A:** `Service` + `AggregateOffer`
  (`lowPrice`/`highPrice`/`offerCount`, `priceCurrency: VND`), числа авто-считаются
  из `prices.json` в `build.py`. Отклонены: полный `OfferCatalog` (сотни строк
  JSON-LD ×10 услуг, шум) и `priceRange` текстом (теряются конкретные low/high).
- **Контент — качественный**, без выдуманных чисел (как три категории).
- Структурные правки шаблона услуг применяются ко **всем 10 услугам** сразу
  (крошки, `Service`-узел, `alt`); видимый FAQ-контент — только у эпиляции
  (guard `{% if svc.faq %}`). Тот же паттерн, что журнал применил к категориям.

## Архитектура

Статический генератор: `generator/build.py` рендерит Jinja-шаблоны в
`vn.neva.beauty/`. Данные — `generator/data/{site,content}.yml` и `prices.json`.
JSON-LD собирается в `generator/schema.py` (Python, не в шаблоне). Общий шаблон
услуги — `service.html.j2`; общий шаблон категории — `category.html.j2` (уже
структурный: крошки + FAQ). Задача — довести `service.html.j2` до того же
структурного уровня и добавить `Service`-узел.

## Юниты (что меняется)

### Структурный слой (влияет на все 10 услуг)

**1. `generator/templates/service.html.j2`**
- **Хлебные крошки** (визуальные), по образцу `category.html.j2`, с учётом
  вложенности:
  - категория-страница (`is_page`): `Главная / <Категория> / <Услуга>`
    (средняя крошка ведёт на `cat.url`);
  - одиночная категория (`is_page = false`, напр. эстетическая косметология):
    `Главная / <Услуга>` (без средней крошки).
- **Блок FAQ** — копия конструкции из `category.html.j2` (`<details>`, первый
  `open`, `{{ item.a | safe }}`), guard `{% if svc.faq %}`, `id="faq"`.
- **`alt` героя**: `«{{ svc.title }} — Neva Beauty, Дананг»` вместо просто
  `svc.title`.

**2. `generator/build.py`** (цикл услуг)
- Прокинуть в шаблон услуги объект категории (сейчас есть только строка
  `svc.category`). Построить `category_by_slug` из `content["categories"]`.
- Прогнать `svc.faq` (если есть) через `render_faq_contacts` — подстановка
  `{whatsapp}/{telegram}/{instagram}` из `site.yml` (как для категорий и главной).
- Собрать per-page узлы для услуги:
  `BreadcrumbList` + `Service(AggregateOffer)` + `FAQPage` (если есть faq),
  поверх базового графа `Organization + BeautySalon + WebSite`.
- Посчитать `lowPrice`/`highPrice`/`offerCount` из ценовых секций услуги
  (`prices.get(slug, [])`): распарсить строки вида `"1 600 000 đ"` в целые VND,
  взять min/max, посчитать число позиций. Вынести в отдельную функцию
  (напр. `price_aggregate(sections)`), чтобы не раздувать `main()`.

**3. `generator/schema.py`**
- Новая функция `service_node(name, description, provider_ref, area_name,
  aggregate_offer=None)` → узел `Service` с `provider` (ссылка на `#business`),
  `areaServed` (City), и, если передан, `offers` = `AggregateOffer`
  (`priceCurrency`, `lowPrice`, `highPrice`, `offerCount`). Описание в узле —
  чистый текст (без HTML), как остальные текстовые поля.

### Контентный слой (только `lazernaya-epilyaciya` в `content.yml`)

**4. `generator/data/content.yml → services.lazernaya-epilyaciya`**
- **seo_title** (сейчас ~40): `Лазерная эпиляция всех зон в Дананге — Neva Beauty`
  (цель 50–60).
- **seo_desc** (сейчас ~74): расширить до 140–160 с ключом, зонами,
  «бережно к коже», мягким CTA и «приём по записи».
- **intro** (hero, answer-first) — заменить маркетинговый текст на
  самодостаточное определение: услуга + бренд + город + суть метода
  (воздействие на корень волоса). **Тип лазера не называть** (данных нет).
- **faq** — новый ключ, 5 реальных Q&A, стиль категорий, с внутренними ссылками
  на родственные услуги и мессенджерами-плейсхолдерами:
  1. Больно ли делать лазерную эпиляцию? — тепло/лёгкое покалывание
     (без «системы охлаждения» — не подтверждена для эпиляции).
  2. Сколько нужно сеансов? — **качественно**: зависит от зоны/типа волос,
     курс подбирают на консультации (без числа).
  3. Какие зоны можно обработать? — список реальных зон из прайса.
  4. Сколько стоит лазерная эпиляция в Дананге? — диапазон + отсылка к «Ценам».
  5. Как записаться? — WhatsApp/Telegram/Instagram (плейсхолдеры).
