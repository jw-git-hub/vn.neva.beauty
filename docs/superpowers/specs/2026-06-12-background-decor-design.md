# Фоновый декор + скролл-движение — дизайн

Дата: 2026-06-12
Проект: vn.neva.beauty (standalone статика, генератор Python + Jinja2)

## Цель

Фон сайта сейчас — плоская заливка нюдового тона; крупные секции выглядят
пустовато. Добавить сдержанный фоновый декор и движение при скролле, чтобы
страница «жила», не отвлекая от контента и не выглядя дёшево.

## Решения (согласованы с пользователем)

- **Тип декора:** световая пыль / bokeh — мягкие размытые полупрозрачные круги
  в тон палитры (`--rose-soft`, `--taupe`). Абстрактно, премиально.
- **Охват:** акцентно по секциям (там, где фон пустует), не сплошным слоем.
- **Движение:** мягкие появления секций/карточек (fade-up) + лёгкий параллакс
  частиц декора.
- **Параллакс на мобиле:** включён везде. Если на телефонах появится джиттер —
  отключаем точечно (флаг / media-query). Архитектура должна это позволять дёшево.
- **Reveal на прайс-строках:** НЕ анимируем (длинные таблицы → назойливо).
  Reveal только на заголовках секций и плитках-карточках.
- **Доступность:** всё движение выключается под `prefers-reduced-motion: reduce`;
  контент при этом виден сразу (без «пустого» экрана).

## Архитектура

Сайт — статический генератор: `generator/build.py` рендерит Jinja-шаблоны в
`vn.neva.beauty/`. CSS и JS лежат напрямую в `vn.neva.beauty/assets/` и являются
источником истины (build их не трогает). Базовый шаблон `base.html.j2` — общий
для всех страниц.

### Юниты

**1. `generator/templates/partials/decor.html.j2`** — Jinja-макрос `decor(variant)`
- Отдаёт `<div class="decor decor--{variant}" aria-hidden="true">` c 4–6 спанами
  `<span class="decor__dot" data-depth="…">`.
- `variant` задаёт раскладку/размеры частиц для конкретной секции
  (напр. `hero`, `soft`, `accent`).
- Единственный источник разметки частиц — DRY между home и service.
- Зависит от: классов в `decor.css`.

**2. `vn.neva.beauty/assets/css/decor.css`** — визуальный слой + классы движения
- `.decor` — абсолютный слой, `pointer-events:none`, `z-index` под контентом,
  `overflow` секции скрывает выходящие частицы.
- `.decor__dot` — круг через `radial-gradient`, `filter: blur()`, низкая
  непрозрачность; размер/позиция через CSS-переменные или модификаторы.
- `@keyframes drift` — медленное «дыхание»/дрейф частиц.
- `[data-reveal]` — начальное состояние (прозрачность 0, смещение вниз),
  `.is-visible` — конечное; transition fade-up. Stagger карточек через
  `transition-delay` по `:nth-child`.
- `@media (prefers-reduced-motion: reduce)` — отключает drift; `[data-reveal]`
  сразу видим; параллакс не применяется (JS проверяет тот же признак).
- Подключается в `base.html.j2` (после `components.css`) → на всех страницах.

**3. `vn.neva.beauty/assets/js/reveal.js`** (ES-модуль) — появления
- `IntersectionObserver` по `[data-reveal]`; при входе в вьюпорт добавляет
  `.is-visible` и отписывается (одноразово).
- Guard: если `prefers-reduced-motion: reduce` — сразу помечает все видимыми,
  observer не создаётся.
- Зависит от: классов в `decor.css`, атрибутов `data-reveal` в шаблонах.

**4. `vn.neva.beauty/assets/js/parallax.js`** (ES-модуль) — параллакс декора
- Один `scroll`-листенер (`passive`), throttle через `requestAnimationFrame`.
- Каждой `.decor__dot` сдвигает `translate3d(0, y, 0)`, где `y` зависит от
  `data-depth` и позиции скролла.
- Guard: при `prefers-reduced-motion: reduce` модуль не инициализируется.
- Зависит от: разметки `.decor__dot[data-depth]`.

### Точки интеграции в шаблонах

- `base.html.j2`:
  - `<link rel="stylesheet" href="/assets/css/decor.css">` в `<head>`.
  - Подключение `reveal.js` и `parallax.js` как `type="module"` перед `</body>`.
  - Импорт макроса decor доступен в дочерних шаблонах.
- `home.html.j2`: вставка `{{ decor(...) }}` в hero (усилив существующие
  `hero__blob`/`hero__ring`), а также в секции Услуги, benefits, Популярное,
  booking — там, где фон пуст. `data-reveal` на `.section-head` и на
  плитках (`.cat-card`, `.popular-card`, `.benefit`).
- `service.html.j2`: `decor(...)` в шапке и пустых зонах; `data-reveal` на
  `.section-head` и карточках-плитках; прайс-строки НЕ трогаем.

## Поток данных / поведение

1. Браузер грузит статику; CSS задаёт декор и начальное (скрытое) состояние
   reveal-элементов.
2. `reveal.js` наблюдает `[data-reveal]`, по мере прокрутки проявляет их.
3. `parallax.js` на каждом кадре скролла слегка сдвигает частицы по `data-depth`.
4. При `prefers-reduced-motion` оба JS-модуля — no-op, CSS показывает контент
   статично.

## Перф и риски

- 4–6 частиц на секцию, только GPU-трансформы (`translate3d`), `will-change`
  точечно на `.decor__dot`.
- `blur()` на больших радиусах дорог — держим радиусы умеренными, количество
  малым.
- Параллакс на мобиле — на контроле; при джиттере отключаем через media-query
  внутри `parallax.js`/CSS-флаг (изменение в одном месте).
- Без новых зависимостей; всё на нативных API (IntersectionObserver, rAF).

## Тестирование / проверка

- Пересборка: `python generator/build.py` (venv) без ошибок.
- Прогон `generator/check_prices.py` — прайсы не сломаны.
- Визуальная проверка home + одной service-страницы: декор виден, появления
  срабатывают, параллакс работает, контент читаем.
- Проверка с включённым «reduce motion» — контент виден, движения нет.

## Вне scope (YAGNI)

- Canvas/WebGL-частицы, физика, интерактив на курсор.
- Декор на всю страницу сплошным слоем.
- Анимация прайс-строк.
- 3D/ДНК-мотивы (ранее отклонены).
