# Оптимизация страницы «Эндосфера-терапия» — дизайн

Дата: 2026-07-13
Проект: vn.neva.beauty (standalone статика, генератор Python + Jinja2)
Страница: `/endosfera-terapiya/`

## Цель

Применить к странице услуги «Эндосфера-терапия» оба промта оптимизации
(`promt-1-seo.md` + `promt-2-ai-geo.md`) — следующая строка `[ ]` в
`TODO-optimization.md`. Третья услуга после «Лазерной эпиляции» (довела
шаблон услуг до структурного уровня) и «LPG-массажа» (первая контентная
оптимизация по этому же шаблону). Поэтому работа здесь — **чисто
контентная**, в одном файле `content.yml`.

## Ключевой факт: структурный слой уже готов

`service.html.j2`, `build.py`, `schema.py` после эпиляции сами рендерят:
крошки с учётом вложенности, узел `Service` + `AggregateOffer` (числа из
`prices.json` через `price_aggregate`), блок FAQ (guard `{% if svc.faq %}`,
первый `<details>` открыт), подстановку мессенджеров `render_faq_contacts`.
Правок шаблонов/Python **не требуется** — только `services.endosfera-terapiya`
в `content.yml`.

## Жёсткие ограничения (из промтов)

- **Не менять цены** — диапазон в schema считается автоматически из прайса.
- **Не выдумывать факты.** Реальные факты эндосферы со страницы/прайса: зоны
  **тело** и **лицо**; пакеты **1 / 5 / 10 сеансов** для каждой зоны; метод —
  **компрессионно-микровибрационный аппаратный массаж** (родовое описание
  метода Endosphères — параллель вакуумно-роликовому определению LPG). Нет
  данных о длительности сеанса, противопоказаниях, числах в «расщеплении
  жира» — не пишем.
- **Не менять URL** `/endosfera-terapiya/`.
- Пользовательский контент — на русском; технические ключи — английские.

## Честность формулировки метода

Мировая маркетинговая формулировка метода Endosphères — «компрессионно-
микровибрационный» аппаратный массаж (сжатие тканей роликовой насадкой +
микровибрация), это реальное родовое определение метода, а не выдумка —
прямая параллель тому, как для LPG было использовано «вакуумно-роликовый».
Остальные факты (зоны тело/лицо, пакеты 5 и 10 сеансов + разовый сеанс,
безболезненность аппаратного массажа, отсутствие синяков/реабилитации) уже
были на странице или напрямую следуют из `prices.json` — числа не
выдумывались.

## Что меняется — `content.yml → services.endosfera-terapiya`

Четыре поля + новый ключ `faq`. `benefits`/`related`/шаблоны — не трогаем.

- **seo_title** 41→57 симв.: `Эндосфера-терапия для тела и лица в Дананге —
  Neva Beauty`.
- **seo_desc** 111→157 симв.: компрессионно-микровибрационный метод +
  коррекция тела/омоложение лица + лимфодренаж/целлюлит/отёки + «Приём по
  записи».
- **intro** (answer-first; идёт и в hero, и в `Service.description` как
  чистый текст): определение услуги + бренд + город + метод + зоны тело/лицо
  + разово/курсом, комфортно в жарком климате Дананга.
- **faq** — 5 реальных Q&A (стиль LPG), первый `open`:
  1. Больно ли делать эндосфера-терапию? — аппаратный массаж, безболезненно,
     без синяков и реабилитации.
  2. Сколько нужно сеансов? — курсом; **для тела и для лица пакеты 5 и 10 +
     разовый** (из прайса); точное число индивидуально.
  3. Что можно скорректировать? — тело (силуэт/целлюлит/отёки/тонус) и лицо
     (лимфодренаж/деликатное омоложение); ссылки на `/lpg-massazh/` и
     `/ultrazvukovaya-kavitaciya/`.
  4. Сколько стоит в Дананге? — зоны тело/лицо + 1/5/10 сеансов; «Цены» на
     странице.
  5. Как записаться? — WhatsApp/Telegram/Instagram (плейсхолдеры), приём по
     записи, ru+en.
